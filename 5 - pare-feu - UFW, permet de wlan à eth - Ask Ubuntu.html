UFW, permet de wlan à eth

Poser une question
1

Je voudrais utiliser mon ordinateur portable Ubuntu 12.04 comme hotsport wifi pour mon smatphone. Tout se passe bien, en utilisant dnsmasq et hostapd et en autorisant les ports 68 et 53 sur UFW. Mais pour accéder au Web, l'UFW doit être désactivé. La connexion est correcte (wifi connecté, adresse IP correctement reçue, requête DNS transférée ...). Dans le journal complet UFW, je vois:

Oct  3 17:09:41 ccd-7840l kernel: [28302.397796] [UFW AUDIT] IN=wlan1 OUT=eth8 MAC=8c:...:00 SRC=192.168.0.59 DST=74.125.234.196 LEN=60 TOS=0x00 PREC=0x00 TTL=63 ID=22391 DF PROTO=TCP SPT=46279 DPT=443 WINDOW=14600 RES=0x00 SYN URGP=0 
Oct  3 17:09:41 ccd-7840l kernel: [28302.397841] [UFW BLOCK] IN=wlan1 OUT=eth8 MAC=8c:...:00 SRC=192.168.0.59 DST=74.125.234.196 LEN=60 TOS=0x00 PREC=0x00 TTL=63 ID=22391 DF PROTO=TCP SPT=46279 DPT=443 WINDOW=14600 RES=0x00 SYN URGP=0

Ensuite, j'ai ajouté les règles suivantes:

sudo ufw allow from 192.168.0.59
sudo ufw allow in on wlan1 from  192.168.0.48/28

Mais ça bloque toujours. Je ne sais pas quoi chercher sur google pour résoudre ce problème :)

Cette solution a fonctionné pour moi.

Premièrement, le transfert de paquets doit être activé dans ufw. Deux fichiers de configuration devront être ajustés. Dans / etc / default / ufw, remplacez DEFAULT_FORWARD_POLICY par «ACCEPT»:

DEFAULT_FORWARD_POLICY = "ACCEPT"

Puis éditez /etc/ufw/sysctl.conf et annulez la mise en commentaire: (je suppose que votre script de démarrage du point d'accès virtuel définit déjà le transfert IP. Vous pouvez donc ignorer cette étape. Ou vouloir supprimer la ligne de transfert IP de votre script.)

net/ipv4/ip_forward=1

De même, pour le transfert IPv6, supprimez la mise en commentaire:

net/ipv6/conf/default/forwarding=1

Nous allons maintenant ajouter des règles au fichier /etc/ufw/before.rules. Les règles par défaut ne configurent que la table de filtres. Pour activer le masquage, il est nécessaire de configurer la table nat. Ajoutez ce qui suit en haut du fichier, juste après les commentaires de l’en-tête: (Si vous utilisez un script pour démarrer votre point d'accès, vous pouvez ou devez probablement supprimer toutes les lignes relatives au masqueradaing.)

># nat Table rules
>*nat
>:POSTROUTING ACCEPT [0:0]

># Forward traffic through eth8.
>-A POSTROUTING -s 192.168.0.0/24 -o eth8 -j MASQUERADE

># don't delete the 'COMMIT' line or these nat table rules won't be processed
>COMMIT

Les commentaires ne sont pas strictement nécessaires, mais il est considéré comme une bonne pratique de documenter votre configuration. De même, lors de la modification de l'un des fichiers de règles dans / etc / ufw, assurez-vous que ces lignes constituent la dernière ligne de chaque table modifiée:

# don't delete the 'COMMIT' line or these rules won't be processed
COMMIT

Une instruction COMMIT correspondante est requise pour chaque table. Dans ces exemples, seules les tables nat et filter sont montrées, mais vous pouvez également ajouter des règles pour les tables raw et mangle. [Remarque]

Dans l'exemple ci-dessus, remplacez eth8 et 192.168.0.0/24 par les interfaces et la plage IP appropriées pour votre réseau.

Enfin, désactivez et réactivez ufw pour appliquer les modifications:

sudo ufw disable && sudo ufw enable

Le masquage IP devrait maintenant être activé. Vous pouvez également ajouter des règles FORWARD supplémentaires au /etc/ufw/before.rules. Il est recommandé d’ajouter ces règles supplémentaires à la chaîne ufw-before-forward.
